Non-Negotiable Principles

1. SEO is not a layer. It is the architecture.

Every route, section, module, and content block must be designed with search discoverability in mind.

2. Design must support trust and conversion.

The UI should look polished enough that users feel they are dealing with a real, high-quality brand.

3. Content must not be thin.

No generic filler. No lazy placeholder paragraphs. No meaningless cards. Every page must have enough value to deserve ranking.

4. Reusability is mandatory.

Do not hardcode product pages manually. Design systems, templates, schemas, and rendering pipelines that can scale.

5. JSON-first content management.

Products, categories, FAQs, blogs, SEO metadata, image mappings, and brand-configurable sections should be structured for file-driven management.

6. Future backend compatibility.

Even though there is no backend now, your architecture should not block future integration with databases, CMSs, carts, payment flows, user auth, or admin dashboards.

7. WhatsApp ordering must feel intentional.

The user experience should feel designed around WhatsApp ordering, not like a missing checkout system.

Core stack

	•	Framer Motion only when it improves UX without harming performance
	•	Lucide React for icons

Deployment
	•	Must be deployable cleanly on Vercel

	•	Must avoid unnecessary client-side JavaScript

Rendering philosophy
	•	Prefer Server Components by default
	•	Use Client Components only when necessary
	•	Keep interactivity selective and intentional
	•	Never bloat bundles with needless client rendering

File/content philosophy
	•	Content should be managed through structured files
	•	The architecture should allow easy migration later to CMS or database
	•	Product rendering should come from structured objects, not duplicated JSX

Processing expectation

	•	products.json
	•	categories.json
	•	mapping by imageKey or product slug
